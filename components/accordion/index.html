An accordion is a stacked list of headings where each heading controls a hidden content area. Clicking a heading reveals or hides its related panel. Accordions help reduce scrolling and keep information grouped and easy to scan.

Key terms:

  • Accordion header – The clickable title that opens or closes a section.
  • Accordion panel – The content area linked to its header.

Each accordion section is made of two parts:

  • A heading containing a button, which acts as the interactive control.
  • A panel, which holds the content that expands or collapses.

Important structure notes:

  • The header button controls exactly one panel.
  • The button is the only element inside the heading element.
  • Panels contain the actual text or UI elements that are revealed or hidden.

Key attributes:

  • The header uses a button element (or an element with role="button").
  • The button sits inside a heading (<h2>, <h3>, etc.) or an element with role="heading" and an appropriate aria-level.
  • The button uses:
    • aria-expanded="true|false" to reflect the current state.
    • aria-controls="panel-id" to reference the controlled content.
  • Panels may optionally use:
    • role="region"
    • aria-labelledby="button-id", pointing back to the header.
  • Use role="region" only when it helps clarify structure (avoid overuse in large accordions).

This implementation uses essential keyboard interactions; optional keys can be added later.

Required:

  • Enter / Space
    Toggle the currently focused accordion header (open when collapsed; close when expanded if closing is allowed).
  • Tab / Shift + Tab
    Move focus through all interactive elements in order.

Optional (not implemented yet):

  • Arrow keys to move between accordion headers.
  • Home / End to jump to the first or last header.
